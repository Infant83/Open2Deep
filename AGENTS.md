# OpenProject Automation Memory

- Use OpenProject tools first for project and work package operations.
- Keep results compact: prefer IDs, identifiers, status, assignee, and short summaries over long dumps.
- Before creating, editing, or commenting on a work package, confirm the exact target and intended change unless the user already gave a direct instruction.
- Resolve project types, statuses, priorities, and assignees before mutations.
- If the context is getting long, summarize the important IDs and next actions instead of repeating full tool output.
